# M3G Live Probe to Snapshot Mapping Contract

## Purpose
This document defines how controlled live probe evidence maps to the `mock_inputs` shape expected by the downstream generator `scripts/generate_latest_market_snapshot.py`. This contract specifies the deterministic translation between the evidence artifacts (generated by the bounded controlled runner) and the input parameters required to build the snapshot.

**Important Note**: This mapping contract is implemented in a strict, read-only, offline adapter layer. It does not actively invoke production refresh automation, run live probes, or modify frontend artifacts.

## A. Input Evidence Shape

The primary entrypoint for the mapping is a run summary file:

*   **Filename Format**: `run_summary_<timestamp>.json`
*   **Top-level Fields**:
    *   `timestamp`: (string, ISO-8601) The time the probe run started.
    *   `targets`: (list of strings) The symbols targeted in the run.
    *   `sources_requested`: (list of strings) The sources attempted.
    *   `results`: (dict) A mapping of `source_id` to a source summary object.

*   **Compatibility Note**: This contract intentionally follows the canonical post-M3G-09 dict shape emitted by `scripts/run_m3g04_controlled_live_probe.py`. Earlier array-shaped draft summaries are legacy and must be normalized before use or rejected as malformed input.

*   **Per-Source Summary Fields** (inside `results[source_id]`):
    *   `status`: (string) e.g., 'success', 'failed'.
    *   `contract_status`: (string) e.g., 'normalized_pass', 'identity_mismatch', 'offline_mode'.
    *   `http_ok`: (boolean) Whether the HTTP request succeeded.
    *   `parse_status`: (string)
    *   `normalization_status`: (string)
    *   `failed_targets`: (list of strings) Symbols that failed for this source.
    *   `errors`: (list of strings) Errors encountered at the source level.
    *   `output_file`: (string, optional) Relative path to the detailed evidence file for this source. If missing, it indicates a wrapper-level failure or offline condition.

## B. Referenced Per-Source Evidence File Shape

The `output_file` points to a JSON file containing detailed, per-symbol evidence.

*   **Standard Envelope Fields** (where available):
    *   `source`: (string) Source identifier.
    *   `source_type`: (string)
    *   `contract_status`: (string)
    *   `http_ok`: (boolean)
    *   `parse_status`: (string)
    *   `normalization_status`: (string)
    *   `normalized_sample`: (dict or list) Can be a symbol-indexed map, a list of objects, or a single normalized object (as emitted by current probes like Yahoo).
    *   `failed_targets`: (list of strings)
    *   `unsupported_targets`: (list of strings)
    *   `warnings`: (list of strings)
    *   `errors`: (list of strings)
    *   `freshness_status`: (string)
    *   `delay_status`: (string)
    *   `staleness_seconds`: (integer or null)
    *   `is_usable_now`: (boolean)

## C. Desired Output Shape

The adapter standardizes the varied internal `normalized_sample` shapes into an internal symbol map shape, and ultimately produces an in-memory dictionary matching the `mock_inputs` parameter expected by `build_snapshot()`.

*   **Adapter-internal symbol map shape**: `dict[standard_symbol] -> normalized_data_object`
*   **Final Output Shape**: `mock_inputs: dict[source_id][standard_symbol] -> snapshot_source_data`

Each `snapshot_source_data` dictionary may include the following fields (mapped from the `normalized_sample` and envelope metadata):

*   `name`
*   `exchange`
*   `last_price`
*   `change`
*   `change_pct`
*   `open`
*   `high`
*   `low`
*   `previous_close`
*   `volume`
*   `bid_ask`
*   `source_time`
*   `retrieved_time`
*   `price_semantics`
*   `freshness_status`
*   `delay_status`
*   `staleness_seconds`
*   `data_quality_flags`
*   `caveats`
*   `raw_payload_ref`

## D. Source-Specific Mapping Rules

### TWSE_MIS
*   **Fields**: Map MIS normalized price fields directly into snapshot fields where present.
*   **Bid/Ask**: Preserve bid/ask structure if available.
*   **Classification**: Classify as `live_candidate`.
*   **Caveats**: Add `unofficial_source_risk`.

### Yahoo_Finance
*   **Fields**: Map `regular_market_price` to `last_price`. Map Yahoo timestamps to source/retrieved time fields where possible.
*   **Classification**: Classify as third-party `live_candidate` or stale candidate depending on delay/staleness.
*   **Validation**: Fail closed on `identity_mismatch`.
*   **Caveats**: Add `third_party_coverage_caveats`.

### TWSE_OpenAPI
*   **Classification**: Map only as official EOD reference. Never classify as live intraday.
*   **Caveats**: Add `official_eod_reference_only`.

### TPEx_OpenAPI
*   **Classification**: Map only as official EOD reference. Never classify as live intraday.
*   **Caveats**: Add `official_eod_reference_only`.

## E. Failure Mapping

*   **Identity Mismatch**: Any `identity_mismatch` must completely block the creation of successful `mock_inputs` for the affected source/target.
*   **Failed Summaries**: Failed source summaries must be translated into source health/failed source evidence in the adapter result; they do not yield valid `mock_inputs`.
*   **Target Preservation**: `failed_targets` and `unsupported_targets` must be explicitly preserved in the adapter output.
*   **Semantics**: Stale, EOD, and live candidate semantics must remain transparent and must not be hidden during the mapping. If downstream freshness policy computes staleness beyond its threshold, `stale_quote` may conservatively override `delayed_quote`; this must remain explicit in `freshness_status`, `delay_status`, and `price_semantics` rather than being presented as realtime.

## F. Fail-Closed Rules

1.  **Malformed Summary**: If the top-level run summary is malformed or unreadable, the adapter returns failure (`adapter_status = malformed_input`) and yields no `mock_inputs`.
2.  **Missing Output File**: If the referenced `output_file` for a specific source is missing, the adapter blocks mapping for that specific source. It returns `adapter_status = partial_mapping` (if other sources are valid) and yields no `mock_inputs` for the missing source.
3.  **Identity Mismatch Block**: An `identity_mismatch` prohibits any promotion for the affected source/target.
4.  **Unknown Target Mapping**: If a symbol's form is unknown or unsupported by the local standardizer, the adapter must preserve the failed target and not attempt to guess or infer its identity.
5.  **Strict Read-Only Guarantee**: The adapter is strictly read-only. It must **never** write generated artifacts to `research/generated/*` or `frontend/public/*`.
